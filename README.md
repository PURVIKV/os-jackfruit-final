# Project Overview

This project demonstrates resource monitoring and workload experimentation in Linux using a kernel module and user-space workload generators.

The system consists of:

* A kernel module (`monitor.c`) for monitoring system events
* A user-space runtime (`engine.c`)
* Workload generators that simulate CPU, memory, and IO pressure

The goal is to explore core concepts from operating systems such as:

* Process scheduling
* Resource monitoring
* Kernel module interaction
* System workload behavior

```

```

---

# Scheduling Experiment

To measure CPU execution time:

```
time ./cpu_hog
```

Example output:

```
real 0m9.699s
user 0m9.698s
sys 0m0.013s
```

This experiment demonstrates how CPU-intensive workloads are scheduled by the operating system.

# Clean Up

Remove the kernel module:

```
sudo rmmod monitor
```

Verify removal:

```
lsmod | grep monitor
```
